# The Vault — 발표 대본

> 총 12슬라이드 / 예상 소요 시간: 8~10분

---

## 슬라이드 1 — 표지

안녕하세요. 저는 "The Vault"라는 프로젝트를 발표하겠습니다.
The Vault는 상품 소개부터 테이블 주문까지, 순수 CSS만으로 구현한 멀티 카테고리 쇼케이스 플랫폼입니다.

---

## 슬라이드 2 — 시작 동기

처음 이 프로젝트를 시작한 이유는 단순했습니다.
상품을 소개하고 판매하는 사이트를 하나 만들어보고 싶었습니다.
F1 굿즈나 NFT 컬렉션처럼, 각각 개성이 뚜렷한 상품들을 한 곳에서 보여주는 쇼케이스. 거기서 출발했습니다.

---

## 슬라이드 3 — 확장 과정

그런데 두 개 섹션만으로는 좀 허전했습니다.
그래서 지식 공유 베이스 느낌으로 개발자 도구 사전을 추가했고,
마지막으로 실제 테이블 주문 UX를 실험하기 위해 푸드 섹션을 넣었습니다.
결과적으로 F1, NFT, Food, Knowledge — 4개의 완전히 독립적인 섹션이 하나의 템플릿 위에서 돌아가는 구조가 됐습니다.

---

## 슬라이드 4 — 문제 의식

사실 이 프로젝트에서 가장 하고 싶었던 건 푸드 섹션입니다.
실제 식당에서 테이블 주문 앱을 써보신 적 있으실 겁니다.
상품 사진을 누르면 모달이 뜨고, 상세설명을 읽고, 장바구니에 넣고, 모달을 닫고... 터치가 3~4번 필요합니다.

저는 이걸 터치 한 번으로 줄이고 싶었습니다.
모달 대신 카드가 바로 펼쳐지면서 상세정보가 나오는 방식.
화면 전환 없이 맥락을 유지하는 거죠.

---

## 슬라이드 5 — 해결: 펼침 카드

이게 그 결과입니다.
4x2 그리드에서 카드 하나를 탭하면 — 또는 데스크톱에서는 마우스를 올리면 — 해당 카드가 인라인으로 확장됩니다.
같은 행의 다른 카드는 자연스럽게 줄어들고 어두워집니다.
펼쳐진 카드에는 제품명, 설명, 가격, 링크가 바로 보입니다.
모달 없이, 페이지 전환 없이, 한 번의 인터랙션으로 끝나는 UX입니다.

---

## 슬라이드 6 — 벤토 그리드

전체 레이아웃은 Apple의 제품 페이지에서 영감을 받았습니다.
Hero 카드가 크게 자리잡고, 오른쪽과 아래에 보조 카드가 배치되는 벤토 그리드 구조입니다.
4개 섹션이 모두 동일한 6카드 템플릿을 공유하고, 탭을 전환하면 색상 테마만 자동으로 바뀝니다.
F1은 레드, NFT는 바이올렛, Food는 오렌지, Knowledge는 블루입니다.

---

## 슬라이드 7 — CSS 상태 머신

그러면 JavaScript 없이 이 모든 인터랙션을 어떻게 구현했는가.
핵심은 CSS를 상태 머신으로 쓰는 겁니다.

HTML의 radio input이 상태를 저장합니다.
label을 클릭하면 상태가 전이됩니다.
그리고 :checked 선택자와 형제 선택자 ~로 상태에 따른 UI 변화를 CSS가 처리합니다.

이 세 줄의 패턴이 뷰 전환, 테마 교체, 카드 확장 등 모든 인터랙션의 기반입니다.

---

## 슬라이드 8 — CSS 모달

Food 섹션에는 주문 확인 모달도 있습니다.
이것도 같은 원리입니다. checkbox가 체크되면 모달 오버레이가 나타나고,
닫기 버튼은 같은 checkbox의 label이라서 클릭하면 체크가 해제되면서 모달이 닫힙니다.
JavaScript의 addEventListener나 setState 같은 건 한 줄도 없습니다.

---

## 슬라이드 9 — Zero JS

정리하면, 이 프로젝트에 JavaScript는 0줄입니다.
뷰 전환, 테마 교체, 제품 카드 펼침, 주문 모달, 로딩 화면 — 전부 HTML input과 CSS 선택자만으로 구현했습니다.

---

## 슬라이드 10 — 반응형

당연히 반응형도 지원합니다.
데스크톱 1440px에서는 3열 벤토 그리드,
태블릿 1024px에서는 2열,
모바일 768px 이하에서는 1열 스택으로 자동 전환됩니다.
clamp() 함수와 미디어 쿼리만으로 처리했습니다.

---

## 슬라이드 11 — 향후 목표

이 프로젝트는 여기서 끝이 아닙니다.
지금은 CSS 프로토타입이지만, 궁극적으로는 실제 테이블 주문 앱으로 발전시키고 싶습니다.

모바일에서 호버 대신 탭으로 카드가 펼쳐지게 하고,
장바구니와 수량 선택 기능을 추가하고,
백엔드를 연동해서 실제 주문까지 처리하는 앱.

핵심 차별점은 변하지 않습니다.
모달 없는 인라인 펼침 UX.
터치 한 번으로 끝나는 주문 경험.

---

## 슬라이드 12 — 마무리

모달을 없애면 터치 한 번이 줄고,
터치 한 번이 줄면 경험이 바뀝니다.

감사합니다. 라이브 데모는 하단 링크에서 확인하실 수 있습니다.

> **팁:** 질문이 들어올 경우 대비

> - "실제 주문은 어떻게?" → 현재는 프로토타입이고, 백엔드 연동 시 API 호출로 대체될 부분입니다.
> - ":has()는 브라우저 지원?" → Chrome 105+, Safari 15.4+, Firefox 121+ 모두 지원합니다.
