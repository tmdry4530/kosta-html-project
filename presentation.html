실제 테이블 주문 UX를 실험하기 위해 추가한 섹션.

문제 의식

식당 테이블 주문 앱,
왜 이렇게 불편할까?

기존 방식

사진 탭 → 모달 열림 → 상세설명 → 장바구니 추가 → 모달 닫기. 터치 3~4회, 화면 전환 2번.

제안하는 방식

사진 탭 → 카드가 인라인으로 펼쳐짐 → 바로 확인. 터치 1회, 화면 전환 0번.

해결 — 펼침 카드

모달 대신 펼쳐지는 카드

터치 한 번으로 카드가 확장되고, 상세 설명과 가격이 바로 보입니다. 화면 전환 없이 맥락을 유지하는 직관적 UX.

디자인 시스템

Apple 스타일 벤토 그리드

Hero(2x2) + Side(1x1) + Bottom(1x1) 6카드 레이아웃. 4개 섹션이 동일한 템플릿을 공유하고, 색상 테마만 자동 전환됩니다.

기술 — CSS 상태 머신

JavaScript 없이
어떻게 가능한가?

/* 상태 저장 → input */ <input type="radio" id="view-f1" checked> /* 상태 전이 → label 클릭 */ <label for="view-f1">F1</label> /* 상태 반영 → :checked + 형제 선택자 */ #view-f1:checked ~ .app-shell .view--f1 { opacity: 1; visibility: visible; }

기술 — CSS 모달

checkbox 하나로
모달을 열고 닫는다

/* 체크되면 모달 표시 */ #modal-food-1:checked ~ .app-shell .modal-overlay { opacity: 1; visibility: visible; } /* 닫기 = 같은 체크박스의 label */ <label for="modal-food-1">닫기</label>

제약 조건

0

줄의 JavaScript

  • 4개 뷰 전환 — radio :checked + 형제 선택자
  • 섹션별 테마 자동 전환 — CSS 변수 교체
  • 제품 카드 펼침 — flex transition + :has()
  • 주문 모달 — checkbox :checked + overlay
  • 로딩 화면 — @keyframes + fill-mode

반응형

1440px → 1024px → 768px

데스크톱 3열 → 태블릿 2열 → 모바일 1열. clamp()와 미디어 쿼리로 모든 뷰포트에 대응합니다.
